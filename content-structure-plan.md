# 网站内容组织结构规划

## 1. 目录结构

```
content/
├── _index.md                 # 新首页（欢迎页）
├── cs/
│   ├── _index.md            # CS板块主页
│   └── posts/               # CS文章
│       ├── *.md
├── poetry/
│   ├── _index.md            # 诗歌板块主页（预留）
│   └── posts/               # 诗歌文章（预留）
├── games/
│   ├── _index.md            # 游戏板块主页（预留）
│   └── posts/               # 游戏文章（预留）
├── series/                  # 合集定义
│   └── *.md
├── about.md                 # 关于页面
├── archives.md              # 归档页面
└── search.md                # 搜索页面
```

## 2. 菜单结构

```
首页 (Home) - 新设计的欢迎页
├── CS - 计算机科学内容
├── 诗歌 - 文学创作（预留）
├── 游戏 - 游戏相关（预留）
├── 合集 - 系列文章
├── 标签 - 文章标签
├── 归档 - 文章归档
├── 关于 - 关于我
└── 搜索 - 站内搜索
```

## 3. 内容分类标准

### CS板块
- 编程语言学习笔记
- 算法与数据结构
- 系统运维
- 开发工具与环境
- 技术教程

### 诗歌板块（预留）
- 现代诗
- 古体诗
- 散文
- 随笔

### 游戏板块（预留）
- 游戏评测
- 攻略心得
- 游戏开发

## 4. Front Matter 规范

```yaml
---
title: "文章标题"
date: 2024-01-01T00:00:00+08:00
draft: false
tags: ["标签1", "标签2"]
categories: ["分类"]
series: ["系列名称"]  # 可选，用于合集
pin: false           # 是否置顶（合集用）
hidden: false        # 是否在列表隐藏
summary: "自定义摘要" # 可选
cover:
    image: "封面图路径"
    hidden: false
---
```

## 5. 工作流程

1. **创建内容**: 在对应板块目录创建 `.md` 文件
2. **Front Matter**: 按规范填写元数据
3. **本地预览**: `hugo server -D`
4. **构建部署**: `hugo` 生成静态文件

## 6. Hugo 技术可行性评估

### ✅ 支持特性
- **多板块支持**: Hugo 的 Section 功能完美支持
- **自定义首页**: 可通过 `layouts/index.html` 实现
- **菜单系统**: 内置菜单配置灵活
- **内容聚合**: 支持跨板块内容展示
- **主题继承**: PaperMod 主题可扩展

### ⚠️ 注意事项
- 需要更新 `mainSections` 配置
- 菜单权重需要重新规划
- 合集功能需要适配新结构
