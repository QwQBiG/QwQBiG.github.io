# 内容格式规范

## 1. Front Matter 标准模板

### 文章模板
```yaml
---
title: "文章标题"
date: 2024-01-01T00:00:00+08:00
draft: false
tags: ["标签1", "标签2"]
categories: ["分类"]
series: ["系列名称"]  # 可选，用于合集
pin: false           # 是否置顶（合集用）
hidden: false        # 是否在列表隐藏
summary: "自定义摘要" # 可选，不填则自动提取
cover:
    image: "images/cover.jpg"  # 封面图路径
    hidden: false              # 是否在列表隐藏封面
---
```

### 合集模板
```yaml
---
title: "合集标题"
date: 2024-01-01T00:00:00+08:00
draft: false
pin: true           # 必须设置为 true，用于置顶显示
hidden: false
series: ["合集名称"]
---
```

## 2. 内容分类标准

### CS 板块
- **编程语言**: C++, Python, JavaScript 等学习笔记
- **算法**: 算法题解、数据结构
- **系统运维**: Linux, WSL, 服务器配置
- **开发工具**: IDE, Git, 环境配置
- **技术教程**: 项目教程、技术分享

### 诗歌板块（预留）
- **现代诗**: 自由诗、散文诗
- **古体诗**: 绝句、律诗、词
- **散文**: 随笔、游记、感想

### 游戏板块（预留）
- **游戏评测**: 游戏评价、推荐
- **攻略心得**: 通关攻略、技巧分享
- **游戏开发**: 游戏制作相关

## 3. Markdown 格式规范

### 标题层级
```markdown
# 一级标题 - 文章标题（Front Matter 中已定义，正文不使用）
## 二级标题 - 主要章节
### 三级标题 - 小节
#### 四级标题 - 细分内容
```

### 代码块
- 使用围栏式代码块
- 必须指定语言类型
- 行号默认开启

```markdown
```python
print("Hello World")
```
```

### 图片使用
```markdown
![图片描述](images/example.jpg)
```

### 链接
```markdown
[链接文字](https://example.com)
```

### 列表
```markdown
- 无序列表项
- 无序列表项
  - 嵌套列表项

1. 有序列表项
2. 有序列表项
```

### 强调
```markdown
*斜体文本*
**粗体文本**
***粗斜体文本***
~~删除线文本~~
```

## 4. 文件命名规范

### 文章文件
- 使用英文或拼音
- 采用驼峰命名或短横线连接
- 示例：`FanWeiForXunHuan.md`, `cpp-tutorial.md`

### 图片文件
- 使用小写字母和数字
- 使用短横线连接
- 示例：`cover-image.jpg`, `diagram-01.png`

### 目录结构
```
content/
├── cs/
│   ├── _index.md          # 板块主页
│   └── posts/             # 文章目录
│       ├── article-01.md
│       └── article-02.md
├── poetry/
│   ├── _index.md
│   └── posts/
├── games/
│   ├── _index.md
│   └── posts/
└── series/
    └── series-name.md     # 合集定义
```

## 5. 内容质量标准

### 文章要求
- 标题简洁明了，不超过 30 字
- 正文段落不宜过长，适当分段
- 技术文章需包含代码示例
- 配图清晰，有适当说明

### 标签使用
- 每篇文章 2-5 个标签
- 标签应具体且相关
- 避免过于宽泛的标签

### 合集要求
- 系列文章应有明确的主题
- 合集页面需包含系列介绍
- 文章按顺序排列

## 6. 工作流程

### 创建新文章
1. 在对应板块 `posts` 目录创建 `.md` 文件
2. 按模板填写 Front Matter
3. 编写正文内容
4. 本地预览：`hugo server -D`
5. 检查无误后构建：`hugo`

### 创建合集
1. 在 `series` 目录创建合集定义文件
2. 设置 `pin: true`
3. 在文章中添加 `series: ["合集名称"]`

### 内容更新
1. 修改对应 `.md` 文件
2. 更新 `date` 或添加 `lastmod` 字段
3. 重新构建部署

## 7. 兼容性注意事项

- 所有内容需支持深色/浅色主题切换
- 图片使用相对路径
- 避免使用平台特定功能
- 测试移动端显示效果
